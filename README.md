# MGA Wwise Importer

Nuendo から書き出したマーカー／テンポ情報（XML）を Wave に埋め込み、Wwise 等で扱いやすい `_e.wav` を生成する Windows 向けツールです。

## 使い方

| 種別 | 内容 |
|------|------|
| 入力 Wave | 埋め込み先の `.wav`（BWF `bext` の TimeReference をタイムラインオフセットに利用） |
| 入力 XML | Nuendo の tracklist（テンポ／拍子／マーカー／サイクルマーカー） |
| 出力 | 入力と同じフォルダの `{元ファイル名}_e.wav`（既存は上書き） |

`.wav` または `.xml` のどちらかをドロップすると、同フォルダ・同ベース名のペア（例: `a.wav` + `a.xml`）を探して処理します。片方が欠けていればエラーになります。

---

## 埋め込みの概要

```text
入力 a.wav + a.xml
  │
  ├─ bext TimeReference でタイムライン位置を補正
  ├─ テンポ／拍子（＋半端小節・-L/-R 境界）で波形全体をリージョン分割
  │     comment = Txxx-n/d （必要に応じて -L / -R / -A）
  ├─ Name が -L / -R のサイクルを追加リージョン化
  │     comment = -L or -R
  └─ 単発マーカーを埋め込み
        name = マーカー名 / comment = Txxx-n/d
  │
  └─ 出力 a_e.wav（cue + LIST adtl、文字列は CP932）
```

書き込み形式は WAV の `cue` + `LIST adtl` です。

| 種別 | 書き込むチャンク |
|------|------------------|
| リージョン | `labl`（表示名）+ `ltxt`/`rgn`（範囲。テキストは `labl` と同じ） |
| マーカー | `labl`（マーカー名）+ `note`（テンポ／拍子コメント） |

文字列は多くの Wave エディタ向けに **CP932（Shift_JIS）** で書き込みます。リージョンに `note` は使いません（表示名として `labl` を使うため）。

位置はすべてプロジェクト時間（PPQ）→ サンプルへ変換し、`bext TimeReference` を差し引いて **Wave 先頭基準** にします。波形範囲外のマーカー／サイクルは埋め込まず、ログに `[警告]` を出します（切り詰め埋め込みはしない）。

---

## テンポ／拍子リージョン

波形先頭〜終端を隙間なく覆うリージョンです。一定テンポ区間やランプ区間の内部は小節線では切りません。

### 分割境界

| 境界 | 内容 |
|------|------|
| 波形先端・終端 | 必須 |
| テンポ変化 | BPM ジャンプ、およびランプの開始／終了 |
| 拍子変化 | 拍子トラックの変化点 |
| 半端小節（アウフタクト） | 波形先頭／終端が小節途中のとき。先頭側はコメント末尾に `-A` |
| `-L` / `-R` の開始／終了 | ここでテンポ／拍子リージョンも区切る |
| `-L` / `-R` 終了後の半端小節 | 終了が小節途中なら「終了〜次小節線」を別リージョン（`-A`）にし、その後に新規リージョンを開始 |

**`-L` / `-R` 範囲の内部**では、テンポ変化・拍子変化があっても切りません（1 区間にまとめ、等価 BPM を付ける）。該当リージョンのコメント末尾には `-L` または `-R` を付けます（例: `T120-4/4 -L`）。

### Nuendo テンポ（`Func`）

| 条件 | 意味 |
|------|------|
| `Func = 1` | 直前イベント BPM → このイベント BPM へ直線変化（ランプ） |
| `Func` なし | ジャンプ（または同一 BPM の維持） |

ランプは「直前イベント〜`Func=1` のイベント」を 1 リージョンとします。

### コメント書式

```text
T{BPM}-{分子}/{分母}
```

例: `T120-4/4` / `T87-6/4` / `T60-4/4 -A` / `T100-6/4 -L`

- BPM は整数に四捨五入した **等価平均 BPM**（`拍数 × 60 × sampleRate ÷ sampleLength`。区間の実尺に合う一定テンポ）
- 拍子はリージョン開始位置で有効な値
- 表示名として Name（`labl`）へ書き込む（`ltxt` にも同じ文字列）

## 追加のマーカー／リージョン

### `-L` / `-R`（サイクルマーカー）

名前が `-L` または `-R`（前後空白は trim）のサイクルマーカーだけを、テンポ／拍子リージョンとは別に追加リージョンとして書き込みます。位置が重なっても構いません。それ以外の名前のサイクルは現状書き込みません。

| 名前 | 意味 |
|------|------|
| `-L` | ループ指定 |
| `-R` | リムーブ（除外）指定。その範囲の波形を最終成果物から取り除く意図 |

現状の責務は WAVE への埋め込みまでです。将来はこれらの境界で波形をスプリットする想定です。

### 単発マーカー

| 項目 | 内容 |
|------|------|
| 名前（`labl`） | Nuendo 上のマーカー名 |
| コメント（`note`） | その位置の `T{BPM}-{分子}/{分母}`（時点 BPM。ランプ中は補間） |
| 位置 | マーカー Start（オフセット後） |

## ソース構成

| パス | 役割 |
|------|------|
| `Program.cs` | エントリポイント |
| `UI/` | フォーム・ウィンドウ／INI 設定 |
| `Processing/` | ドロップ処理・外部エディタ起動 |
| `Nuendo/` | tracklist XML・テンポ／拍子マップ |
| `Wave/` | WAV 解析、cue 構築（`WavMarkerEmbedder`）、書き込み（`WavCueWriter`） |

名前空間はいずれも `MgaWwiseImporter`（単一プロジェクトのためフォルダと分けない）。

## 開発メモ

- `test/` はローカル検証用でリポジトリには含めません（`.gitignore`）

### `MgaWwiseImporter.ini`（`[Developer]`）

exe と同じフォルダに置きます（無ければ起動時に既定値で作成／不足キーを追記）。`[Window]`（位置・サイズ）とは独立です。

| キー | 意味 | 既定 |
|------|------|------|
| `ExternalEditorPath` | 出力後に開く外部 Wave エディタの実行ファイル | 空（起動しない） |
| `OpenInExternalEditor` | 出力後に外部エディタで開く（`1`/`0`） | `1` |
| `AutoLoadWavePath` | 起動時の自動読み込みパス（空ならスキップ）。相対パス可 | リポジトリの `test\a.wav`（無ければ空） |
| `TopMost` | ウィンドウを最前面に固定（`1`/`0`） | `0`（不足キー追記時は現状デバッグ用に `1`） |

```ini
[Developer]
ExternalEditorPath=C:\Path\To\YourWaveEditor.exe
OpenInExternalEditor=1
AutoLoadWavePath=D:\GitHub\MGA-Wwise-Importer\test\a.wav
TopMost=1
```
